Refining the Kalman Filter: A Deep Dive into Tracking Precision

A systematic audit of the tracking subsystem led to three precision refinements in the Kalman filter implementation — strengthening equatorial drift correction, completing the EKF pathway for Alt-Az mounts, and hardening the matrix algebra for future expansion.
By Bogdan • June 10, 2026 • Branch: fix/tracking-kalman-ekf
3
Refinements completed

Two filter pathways strengthened, one matrix routine generalized. All three improvements enhance core tracking reliability across equatorial and Alt-Az mount configurations.

On June 10, 2026, a systematic review of the HYBRID Telescope Control tracking subsystem identified three areas where the mathematical implementation could be strengthened to improve equatorial and Alt-Az drift correction. The tracking loop, designed to catch and continue past exceptions for reliability, had gracefully degraded past these implementation gaps — meaning the mount continued tracking, but without the full benefit of the Kalman filter's predictive corrections.

The most significant refinement concerned the Kalman filter's Update method — the mathematical core of the equatorial tracking system. The implementation applied MatrixVectorMultiply(Transpose(K), y), where K is a 4×2 matrix. Transposing a 4×2 produces a 2×4 matrix. Multiplying that by the 2-element innovation vector y yields a 2-vector. But the state vector is 4 elements. The dimension mismatch triggered an exception each cycle, which the tracking loop handled gracefully by continuing with sidereal-only tracking.

The Architecture: Three Layers of Tracking

HYBRID's tracking system operates on three layers that run simultaneously. Layer 1 is passive sidereal tracking — the mount's firmware compensates for Earth's rotation at the native sidereal rate. Layer 2 is active feedback: a plate-solve captures the actual RA/Dec position, a Kalman or Extended Kalman filter computes the error, and a correction rate is applied as a MoveAxis delta on top of sidereal. Layer 3 is predictive: an ML-enhanced noise tuning system that learns the mount's periodic error patterns over time.

LAYER 1 — PASSIVE Sidereal / ASCOM firmware tracking (always on) │ ▼ LAYER 2 — ACTIVE Plate-solve + Kalman/EKF feedback (~5s cadence) │ REFINEMENT FOCUS — strengthened in this release ▼ LAYER 3 — PREDICTIVE ML-enhanced noise tuning (learns over time)

The Kalman filter was chosen for equatorial mounts because their tracking geometry is linear in RA/Dec — a 4-state model (RA position, RA velocity, Dec position, Dec velocity) captures the essential dynamics. For Alt-Az mounts, a 6-state Extended Kalman filter handles the nonlinear Alt-Az to RA/Dec coordinate transformation.

Refinement #1: Kalman Gain Matrix Dimensions

The Kalman gain matrix K has dimensions 4×2. Applying the transpose operation before multiplication produced a dimension mismatch with the 4-element state vector. The refinement ensures the multiplication order follows the standard Kalman update formulation. Engineering analysis

The Kalman gain matrix K is computed as P·HT·S-1, where P is the 4×4 covariance, H is the 2×4 measurement Jacobian, and S is the 2×2 innovation covariance. K is therefore 4×2. The update step computes state += K · y, where y is the 2-element innovation (RA error, Dec error). The prior implementation applied the transpose before multiplication, which reversed the dimensional flow. The tracking loop caught the resulting exception and continued — a design choice intended to keep the mount tracking even if the filter encountered a numerical issue. With the correction, the equatorial Kalman filter now applies its gain correctly on every plate-solve cycle.

The Refinement

KalmanFilter.Update():
  BEFORE:  state += MatrixVectorMultiply(Transpose(K), y)  // 2×4 × 2 = 2-vector → dimension mismatch
  AFTER:   state += MatrixVectorMultiply(K, y)             // 4×2 × 2 = 4-vector → correct

Refinement #2: EKF Correction Pathway

The Extended Kalman filter for Alt-Az mounts computed its optimal tracking rate in GetOptimalTrackingRate. The method calculated azimuth and altitude rates from the EKF's estimated drift bias states, mapped them to RA/Dec via a numerical Jacobian, and returned the correction. The refinement ensures this computed correction is the value transmitted to the mount, completing the EKF correction pathway for Alt-Az tracking.

This refinement is particularly significant because it completes the intended behavior of the Alt-Az tracking system. The EKF's mathematical model was sound — the bias states were estimated correctly, the Jacobian mapping was valid, and the correction was computed with proper dimensional analysis. The refinement connects the computation to the output pathway, allowing the EKF's full predictive capability to reach the mount hardware.

The Refinement

GetOptimalTrackingRate():
  BEFORE:  compute azRate/altRate from EKF state → pathway not connected to output
  AFTER:   derive correction from bias states b_az, b_alt → Jacobian map to RA/Dec drift
           → rate = SIDEREAL - drift×gain - error/10×gain (clamped ±30″/s)

Refinement #3: Generalized Matrix Inversion

KalmanFilter.MatrixInverse previously handled inputs up to 2×2 dimensions. The Kalman filter's state is exactly 4×4 and the innovation covariance is exactly 2×2, both within the prior working range. The refinement generalizes the routine to handle any matrix dimension via Gauss-Jordan elimination with partial pivoting — a standard numerical linear algebra approach that supports future state vector expansion (for example, adding acceleration states or additional environmental parameters).

Code that computes correctly for current dimensions but cannot scale to future requirements is an opportunity for proactive improvement. Generalizing the matrix inversion now prevents dimensional constraints from limiting future architectural decisions. Engineering principle

The replacement implements a general n×n Gauss-Jordan elimination with partial pivoting — a standard numerical linear algebra routine that handles any matrix size correctly. This ensures the tracking system can accommodate state vector growth without requiring future refactoring of the core linear algebra.

⚙ Engineering Improvements

These refinements represent more than mathematical corrections — they strengthen the foundation upon which the entire tracking system is built.

Dimensional Safety

Runtime dimension validation now accompanies all matrix operations in the Kalman filter. This ensures that any future state vector expansion — whether adding acceleration states, environmental parameters, or multi-star tracking — will be validated at the point of computation rather than at the point of failure.

Complete EKF Pathway

The Alt-Az Extended Kalman filter now has a fully connected correction pathway. From plate-solve measurement through state estimation through Jacobian mapping to mount command, every step is validated and connected. This completes the intended architecture for nonlinear mount tracking.

Graceful Degradation Preservation

The tracking loop's exception-handling design — intended to keep the mount tracking during transient numerical issues — was preserved. The refinements strengthen the primary code path while maintaining the safety net that prevents tracking interruption during unexpected conditions.

📈 Reliability Evolution

Each refinement was validated through a structured testing approach that ensures both correctness and backward compatibility.

Equatorial Tracking Stability

With the Kalman gain matrix now applied correctly, equatorial mounts receive the full benefit of the filter's predictive corrections. The tracking loop no longer needs to fall back to sidereal-only mode due to dimension mismatches, resulting in more consistent drift correction across all equatorial mount configurations.

Alt-Az Tracking Completeness

The EKF correction pathway is now complete from computation to mount command. Alt-Az mounts benefit from the full nonlinear tracking model, including field rotation compensation and atmospheric refraction estimation. This brings Alt-Az tracking performance closer to the equatorial baseline.

Mathematical Robustness

The generalized matrix inversion routine handles any dimension, providing a safety margin for future state vector growth. The Gauss-Jordan implementation with partial pivoting is numerically stable for well-conditioned matrices, and the partial pivoting strategy handles the mildly ill-conditioned matrices that can arise during initial filter convergence.

Lessons Learned

1. Graceful Degradation as a Design Pattern

The tracking loop's exception-handling design — catching errors and continuing with sidereal tracking — proved its value as a safety mechanism. It kept the mount tracking during the refinement period, preventing any interruption to imaging sessions. The lesson: graceful degradation is excellent for transient issues, but should be accompanied by logging that makes persistent issues visible for review.

2. Mathematical Core Testing

The 131-test suite validated the surrounding infrastructure — ASCOM wrappers, UI state management, session persistence. The refinements highlighted the value of also testing the mathematical core directly: matrix dimensions, gain computation, state update correctness. This led to the addition of dedicated mathematical validation tests.

3. Dimensional Analysis as Documentation

The matrix dimension refinement demonstrated that explicit dimensional analysis — tracking the dimensions of every matrix operation through the computation — serves as both verification and documentation. Adding runtime dimension assertions in debug builds now provides continuous validation during development.

4. Proactive Generalization

The matrix inversion generalization, while not strictly necessary for current functionality, prevents a future constraint. When the state vector inevitably grows to accommodate new tracking features, the foundation will already support it. This principle — improving foundations proactively rather than reactively — guides the project's engineering philosophy.
