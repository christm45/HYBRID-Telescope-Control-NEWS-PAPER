HYBRID Telescope Control is a Windows application that lets you operate an amateur telescope by talking to it. Say "slew to M31, plate solve, and start imaging for 90 minutes" and the scope does it. It is not a replacement for NINA or SGP — those are mature and excellent. It is a fundamentally different interaction model: voice-first control combined with AI-powered tool orchestration via the Model Context Protocol, making it the first amateur astronomy app to integrate MCP for AI-driven telescope control.

The project began in late 2024 as a solo effort by Bogdan, an amateur astronomer and developer based in France. By June 2026, it had grown to 200+ C# source files, 347+ unit tests, 47 MCP tools, 55+ voice actions, and a 9-commit security hardening campaign. This is the story of how it got there.

The Origin: Why Voice?

The observation that started everything was simple: interacting with telescope software at -5°C with gloves on is miserable. Every click requires pulling off a glove, finding the right menu, and clicking a small target — then putting the glove back on before fingers go numb. Voice control solves this not as a convenience but as a genuine usability improvement for the observing workflow.

But voice control alone is not enough. A wake-word system that responds to "Hey Telescope" is a toy — it can handle "go to M31" but not "slew to M31, plate solve, center it, autofocus, and start imaging for two hours with 180-second exposures." The real value is in chaining commands, and that requires an AI that can reason about the sequence. The Model Context Protocol, introduced by Anthropic in 2024, provided the missing piece: a standard way for an AI assistant to discover and call telescope tools.

The Technology

The application is built on three technology layers. The main application is C# targeting .NET Framework 4.8 with WPF for the UI — 200+ source files organized into a 7-project Visual Studio solution. The voice pipeline runs in Python 3.11 (bundled in the installer) because faster-whisper, Silero VAD, and gTTS have no C# equivalents with comparable performance. The MCP server runs on .NET 8, separate from the main application, exposing 47 tools via the Model Context Protocol.

Engineering Milestones and Refinements

Audit Campaign Summary

9 commits across 4 subsystems

2 hardening areas — web auth, meridian flip

3 improvements — process lifecycle, session persistence, FITS handling

1 refinement — UI responsiveness, zenith rate

The most instructive engineering moments were not system failures — they were graceful degradation patterns that masked persistent implementation gaps. The Kalman filter's matrix dimension issue was handled by the tracking loop's exception handling, which kept the mount tracking but without the filter's corrections. The EKF computed its optimal rate but the output pathway was not fully connected to the mount command layer. The ML layer's training pipeline was architecturally complete but needed extension from single-axis to dual-axis regression.

All three refinements shared a common pattern: the system continued to function, tests passed, and no exceptions surfaced to the user. The graceful degradation design — intended to keep the mount tracking during transient numerical issues — meant that implementation gaps were invisible in normal operation. The audit process identified these opportunities for improvement through systematic code review and mathematical validation.

Graceful degradation is excellent engineering for transient issues. Systematic review is the complementary practice that identifies persistent opportunities for improvement. Engineering principle

Security Hardening

The web authentication hardening was the most significant security improvement. The AuthEnabled = false default was appropriate for local development — during debugging, typing a password for every API call is unnecessary friction. The hardening ensures that when the ngrok tunnel is started for remote access, authentication is automatically activated — a secure-by-default approach that eliminates the gap between development convenience and deployment security.

The meridian flip hardening added verification layers for physical safety. The scheduler now verifies pier side changed and halts the session if verification fails — a necessary evolution for unattended observatory operation where physical safety must be guaranteed.

What Voice Control Actually Solves

After 18 months of development and real-world use, the honest assessment is that voice control genuinely helps with three scenarios. First, the 90% case: goto + plate solve + start imaging. You say it once, the scope does it, you walk away. Second, cold-weather operation: changing exposure, filter, or target mid-sequence without touching the laptop. Third, accessibility: observers with mobility limitations or those doing public outreach with groups around a scope gain a more natural interaction model.

Voice is less suited for precise numerical input, complex filter trees, and noisy environments. It is not a replacement for click-based interaction — it is a complement that handles the cases where hands-free is genuinely better.

Lessons Learned

Ship Early, Review Deep

v1.2.0 shipped with real functionality. The audit campaign identified refinements. You can't find what you don't look for — but you can't improve what you don't ship.

Defaults Are Security Decisions

AuthEnabled = false was a development convenience. The hardening established secure-by-default as the architectural principle.

Physical Safety Requires Verification

Hardware commands must verify their effect. "The flip probably worked" is not acceptable when the alternative is equipment damage.

Solo Dev, Structured Review

18 months, one developer, 200+ files. The dev trace log proved invaluable for the audit. Even solo projects benefit from structured review processes.

What's Next

The roadmap includes comment-mode surgical edits (click on any part of a generated artifact and ask for changes), a skill marketplace (od skills install), one-command initialization (npx od init), and a packaged Electron build with bundled sidecars.
