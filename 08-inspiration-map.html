Projects referenced

Telescope control, AI tools, voice recognition, image processing, web frameworks — each left its mark on HYBRID's design.

Every significant software project is a conversation with its predecessors. The developers who built NINA, SGP, and PHD2 solved problems that HYBRID's developer studied before writing a single line of code. The designers of the Model Context Protocol created a standard that made AI-driven telescope control possible. The authors of faster-whisper and Silero VAD built the voice recognition stack that HYBRID's voice pipeline depends on. This is the genealogy — where each idea came from, what was borrowed, and what was invented.

Telescope Control Software

N.I.N.A.
Telescope Sequencing

The gold standard for free, open-source telescope sequencing. HYBRID borrowed the concept of equipment profiles (save/load telescope+camera+filter configs) and the sequencer's target-based workflow. NINA's plugin architecture showed that a telescope app could be extensible without being monolithic.

Sequence Generator Pro
Professional Acquisition

SGP's reliability and polished UX set the bar for what a commercial telescope control app should feel like. HYBRID's session management (auto-save every 30 seconds, crash recovery) was inspired by SGP's robust state persistence. SGP's meridian flip logic was studied for the pier-side verification approach.

PHD2
Autoguiding

The dedicated autoguiding solution that every imager uses. HYBRID integrates with PHD2 for dither settle and guide-star-lost detection, but also includes its own Kalman-based tracking as an alternative. PHD2's approach to guide star selection and dither patterns influenced HYBRID's tracking design.

Stellarium
Sky Chart

HYBRID embeds the Stellarium Web Engine (WASM, BSD 3-Clause) as its in-app sky chart. The real-time sky view, DSS deep-sky overlay, and reticle tracking are all Stellarium technology. The sky chart is HYBRID's most visible borrowed component.

ASCOM Alpaca
Hardware Protocol

The ASCOM Platform is the universal standard for telescope hardware control. HYBRID's ASCOMControllerWrapper is a complete Alpaca HTTP REST client — mount, camera, focuser, rotator, filter wheel, dome, switch, safety monitor. ASCOM made HYBRID possible; without it, every mount vendor would need a custom driver.

ASTAP
Plate Solving

Han Kleijn's ASTAP is HYBRID's primary local plate solver. Fast, offline, reliable. HYBRID also supports Astrometry.net for cloud solving. ASTAP's hinted-solve mode (using the mount's approximate position) was adopted for speed.

AI and Voice Tools

Claude Desktop (Anthropic)
AI Assistant + MCP

The Model Context Protocol is Anthropic's open standard for connecting AI assistants to external tools. HYBRID's 47 MCP tools follow this standard exactly. Claude Desktop is the primary voice backend — it reasons about telescope commands and chains MCP tool calls naturally.

faster-whisper (SYSTRAN)
Speech-to-Text

The CTranslate2 port of OpenAI's Whisper model. HYBRID uses the small.en variant for near-real-time transcription. Silero VAD (from the same ecosystem) handles voice activity detection. Together they form the core of HYBRID's voice input pipeline.

llama.cpp / Ollama
Local LLM

The local LLM backends that make fully offline voice control possible. HYBRID's llama.cpp bridge connects to a local Qwen3 model for remote observatories without internet. Ollama provides a simpler alternative for users who prefer its model management.

VS Code Copilot
AI Code Assistant

HYBRID includes a voice bridge for VS Code Copilot Chat, allowing developers to control the telescope from their editor. The bridge was modeled after Claude Desktop's MCP integration but adapted for Copilot's different protocol.

Image Processing

Siril
Astro Image Processing

The free, open-source astro image processor. HYBRID's stacking algorithms (sigma-clipping, Winsorized, median) were validated against Siril's output. Siril's background extraction (RBF thin-plate spline + polynomial) was studied for HYBRID's BackgroundExtractionEngine. Siril 1.4.2 source was included in build staging for reference.

PixInsight
Professional Processing

The professional standard for astro image processing. HYBRID's SPCC color calibration and drizzle algorithms reference PixInsight's approach. HYBRID's post-processing is positioned as a preview tool — users are expected to take HYBRID's linear masters into PixInsight for final production.

Seti Astro Suite Pro
AI Post-Processing

The original source of the AI Denoise, Star Removal, and Super-Resolution algorithms. HYBRID's numpy implementations were reimplemented from scratch based on SASpro's reference scripts. When Seti Astro Cosmic Clarity (trained PyTorch models) is installed, HYBRID uses those instead.

SharpCap
Live Stacking

SharpCap's live-stacking behavior (Winsorized rejection, automatic background neutralization) was used as a reference for HYBRID's stacking engine. The dev log explicitly notes that HYBRID's winsorize behavior "matches SharpCap."

What Was Invented, Not Borrowed

Voice control for telescope operations is not borrowed from anyone. No prior art exists. The closest analogy is voice-controlled home automation — but controlling a telescope is orders of magnitude more safety-critical than turning on a light. Design reflection

Three features have no clear precedent in the telescope software ecosystem. First, conversational voice control with AI orchestration — no other telescope app offers this. The voice pipeline (microphone → VAD → STT → parser → MCP dispatch → ASCOM command) was designed from scratch because no existing voice integration for hardware control was sophisticated enough to handle astronomical commands.

Second, the 3-layer predictive tracking model (sidereal + Kalman/EKF + ML) is unique. PHD2's guiding is reactive. HYBRID's tracking is predictive — it uses a Kalman filter to estimate the mount's drift state and applies correction preemptively. The ML layer learns periodic error patterns over time. This approach has precedent in aerospace and robotics but not in amateur astronomy.

Third, the UI-freeze watchdog with auto-restart and session recovery is a reliability feature borrowed from industrial control systems, not telescope software. The concept: a background thread monitors the UI thread's responsiveness; if it freezes beyond a configurable threshold (default 45 seconds), the watchdog kills and restarts the application, recovering state from the last auto-save. This is critical for unattended imaging rigs running for 6+ hours.

The Development Stack

Visual Studio 2022

Primary IDE. MSBuild Release|x64 via BUILD.bat. SDK-style .csproj with auto-include. C# 7.3 language version.

NSIS + 7-Zip

Installer pipeline: NSIS compile → stage ~4.9 GB → 7z LZMA2 → SFX. Output: single ~2.64 GB EXE.

MSTest

347+ unit tests across GUI (161), ImageProcessing (357), Camera (34), and WebUI (25 Playwright). Hardware-independent regression tests.

Node.js 18+

Website build (Vite + TypeScript). Server-side analytics (server_with_analytics.cjs). Deployed via SCP/SSH.
