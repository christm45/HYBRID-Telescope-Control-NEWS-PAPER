The HYBRID Telescope Control system does not merely track stars. It listens to your voice, reads the sky, corrects its own errors, and orchestrates every subsystem of an observatory from a single laptop. Forty-eight discrete features — from Kalman-filter predictive tracking to voice commands in two languages — form what is now the most capability-dense amateur astronomy platform ever shipped. This catalog documents every one of them.

Section 1 — Pro Observatory Layer

The standard HYBRID package ships with 41 features covering tracking, imaging, automation, and system management. The Pro tier adds seven capabilities that elevate HYBRID from a telescope controller to a full observatory platform. These are not incremental additions; each one interfaces with external hardware protocols and requires dedicated state management.

Dome Control

Full INDI and Alpaca dome protocol support. The dome shutter, rotation, and home index are tracked in real time. Park, track, and wire modes allow the dome to follow the mount through meridian flips and target switches without manual intervention. Auto sync ensures the dome slit stays aligned with the telescope's optical axis throughout the session.

Power Switch Control

Four-channel relay management with per-switch naming, auto-off timers, and a live power budget display. Each switch can be assigned to a subsystem — mount, camera, dew heater, focuser — and configured with independent on/off schedules. The power budget tracker prevents overloads by monitoring total draw against supply capacity.

Focuser Control

Absolute positioning with backlash compensation in both directions. Auto focus routines use HFR (half-flux radius) measurement to find the critical focus zone. Filter offsets allow instant refocusing when switching between narrowband and broadband filters without repeating the full autofocus cycle.

Rotator Control

Angle synchronization keeps the camera oriented correctly relative to the target throughout long exposures. Meridian flip rotation compensates for the 180-degree image rotation that occurs during pier-side transitions. Framing presets store preferred position angles for common targets.

Calibration Light Source

Flat panel control for calibration frames. The system manages panel brightness, exposure duration, and session planning for dusk/dawn flats. Integration with the sequence engine allows automated flat frames at session start or end.

Polar Alignment

A plate-solve-based routine modeled on the TVC (Takahashi Variation Correction) method. Achieves plus or minus 5 arcsecond polar alignment accuracy through iterative star measurements and mount offset calculations. No additional hardware required beyond the main imaging camera.

Center-and-Rotate

A single command that frames a target by centering it in the field and rotating the camera to the desired position angle. The system plate-solves, calculates the offset, slews the mount, and rotates the camera in sequence — all without user intervention.

Section 2 — Core Tracking Systems (Features 1–12)

The tracking stack is the foundation of everything else. Without sub-arcsecond tracking precision, no image processing or automation matters. HYBRID implements twelve distinct tracking subsystems that work in concert.

Kalman Filter

A 4×4 state-space model tracking RA position, RA velocity, Dec position, and Dec velocity. The filter fuses mount encoder readings with plate-solve corrections to produce a predicted state that smooths out periodic error, backlash, and atmospheric refraction artifacts. An extended Kalman filter variant handles nonlinear Jacobian matrices when tracking rates deviate significantly from sidereal.

OnStepX and INDI Drivers

Direct integration with OnStepX via serial and network connections. INDI drivers provide a universal abstraction layer for mounts, cameras, and accessories. Both paths are supported simultaneously — a user can run OnStepX on a Raspberry Pi while the Windows laptop handles image processing and scheduling.

ASCOM PulseGuide

Standard ASCOM PulseGuide interface with 100-millisecond pulse resolution on both RA and Dec axes. Compatible with PHD2, MetaGuide, and any other guiding application that speaks ASCOM.

Star Catalog

Over 3,000 objects with magnitude, seasonal visibility, and current altitude data. The catalog is queried in real time to suggest targets based on observability windows, moon illumination, and user preferences.

All-Sky Plate Solve

Powered by astrometry.net, the plate solver recovers pointing from any orientation — even a completely lost mount. Solve times range from 2 seconds for well-exposed star fields to 30 seconds for faint-field solutions.

Plate Solve Feedback Loop

A 5-second cadence plate-solve loop that continuously syncs the mount's reported position with the actual sky coordinates. The feedback is fed back into the Kalman filter, creating a closed-loop tracking system that corrects drift in real time.

Periodic Error Correction (PEC)

FFT analysis of tracking errors reveals the worm gear's periodic signature. HYBRID captures the star-shaped error waveform, fits a correction curve, and exports it directly to OnStepX for hardware-level PEC playback. The result is sub-arcsecond tracking without autoguiding.

Section 3 — Voice and AI (Features 13–20)

HYBRID is the first astronomy application to ship production voice control. Not a demo. Not a proof of concept. A working system that handles 210+ phrase variants across 55+ actions in French and English, with ten or more dialect support.

Speech Recognition Engine

OpenAI Whisper large-v3 runs locally on the GPU for sub-second transcription. A Silero VAD (voice activity detection) combined with an energy gate provides dual-layer noise filtering that rejects wind, traffic, and dew heater hum without clipping speech.

Voice Action Coverage

Every major subsystem is voice-controllable: slew to target, start imaging, run autofocus, check weather, park the mount, open the dome. Commands are parsed with fuzzy matching — "point at Andromeda" and "slew to M31" both resolve to the same target.

Three AI Backends

Local processing for zero-latency control. Gemini CLI for conversational target selection and session planning. Claude CLI for complex multi-step commands and natural language scripting. Users can switch backends mid-session based on connectivity and complexity.

MCP Integration

47 tools exposed via Model Context Protocol with OpenAI function calling format. Any LLM that supports function calling can control the telescope — not just the built-in backends.

Section 4 — Image Processing (Features 21–28)

FITS image processing is not an afterthought in HYBRID — it is a first-class citizen. Fifteen distinct analysis methods cover everything from basic statistics to advanced star shape characterization.

Sub-Frame Selector

HFR-based filtering with quality grades A through D. Frames exceeding the user-defined FWHM or eccentricity thresholds are automatically rejected before stacking. This eliminates trailing, bloated stars, and cloud-affected frames without manual inspection.

Background Extraction

Automatic background modeling using mesh interpolation. The extracted background is subtracted from each frame, removing gradient artifacts from light pollution, moon glow, and vignetting.

Live Stacking

Three stacking algorithms: median for maximum outlier rejection, mean for best signal-to-noise, and sigma-clipped mean for a balance of both. Stacks update in real time as new frames arrive, producing a progressively deeper image throughout the session.

Preview Export

8-bit preview images exported as PNG, JPEG, or TIFF for quick sharing. The full 16-bit or 32-bit FITS data is preserved separately for scientific processing.

Section 5 — Session Automation (Features 29–36)

An observatory session is not a single action — it is a sequence of decisions, adjustments, and contingencies that unfold over hours. HYBRID automates the entire workflow.

Multi-Target Scheduler

Targets are queued with altitude constraints, moon avoidance windows, and priority scores. The scheduler reorders the queue in real time as conditions change — a target rising above the horizon takes precedence over one sinking behind trees.

Auto Meridian Flip

When the mount approaches the meridian, HYBRID pauses imaging, verifies pier side, flips the mount, re-centers the target, and resumes — all without user intervention. The plate-solve feedback loop ensures the target is re-acquired within 2 arcseconds of its pre-flip position.

Auto-Guiding

PHD2 integration via ASCOM. Dithering between frames reduces walking noise patterns. Guiding calibration is automated at session start, and guide star selection uses the same HFR metrics as the main imaging pipeline.

Sequence Engine

N-step sequences with loops, conditional logic, and sub-frame selection. A typical sequence might be: 20×300s L, 15×300s R, 15×300s G, 15×300s B, each with autofocus triggers every 10 frames and flat calibration between filter changes.

Safety Monitor

Weather data from OpenWeatherMap and Meteoblue is polled continuously. Wind speed, cloud cover, and humidity thresholds trigger automatic session abort. The safety monitor is independent of the main control loop — it cannot be overridden by voice commands or automation scripts.

Section 6 — System Architecture (Features 37–44)

The remaining eight features in the core package address system reliability, remote access, and integration with the broader home automation ecosystem.

UI Watchdog

A background monitor that detects application freezes and triggers automatic restart. The watchdog preserves session state across restarts — no data is lost when the UI crashes.

Cloud Observatory

Multi-machine architecture allows remote monitoring from any device with a web browser. The laptop next to the mount handles hardware communication; a desktop across the house provides the control interface. Both stay synchronized via WebSocket state replication.

Web Dashboard

Real-time status display — mount position, camera state, weather, guiding error, stacking progress — served from the local machine. No installation required on the viewing device. Works on phones, tablets, and laptops.

SQLite Database

Every session is recorded: target coordinates, exposure times, guiding metrics, weather conditions, and user actions. The database is fully searchable, enabling historical analysis of tracking performance across months of data.

MQTT Integration

Observatory state is published to an MQTT broker for home automation integration. Node-RED, Home Assistant, and custom scripts can subscribe to mount position, session status, and safety alerts.

REST API

Over 60 endpoints expose every subsystem. The API is documented, versioned, and supports both JSON and message-pack serialization. Custom frontends, mobile apps, and third-party tools can control the observatory without touching the HYBRID UI.

Section 7 — Professional Features (Features 45–48)

The final four features target advanced users who operate multi-equipment setups and need enterprise-grade data management.

Frame and Mount Templates

Saved configurations for telescope, camera, mount, and filter wheel combinations. Switching between a wide-field refractor setup and a deep-sky SCT rig takes one click instead of reconfiguring every parameter manually.

Weather Integration

Two weather data sources — OpenWeatherMap for general conditions and Meteoblue for astronomical seeing forecasts. Both are polled at configurable intervals, and the data feeds directly into the safety monitor and scheduler.

Multi-PC Network Control

The daemon, web, and desktop components run on separate machines. The daemon handles hardware communication on the mount-side laptop. The web dashboard runs on any device. The desktop app provides the full Electron experience on a control room machine. State synchronization is automatic and lossless.

Export and Import

Full session data export in JSON format. Import previous sessions for comparison, reproduce imaging runs, or transfer configurations to a new machine. The export includes target metadata, exposure sequences, guiding logs, and weather records.
