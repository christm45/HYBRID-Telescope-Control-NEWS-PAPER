Every telescope mount has a pointing model — a mathematical description of where the mount thinks it is pointing versus where it actually points. For a century, astronomers have built these models by measuring the positions of calibration stars, one by one, across the sky. The result is a polynomial surface that maps desired coordinates to actual mount positions. It works. It is accurate. And it requires hours of manual work that must be repeated whenever the mount moves, the temperature shifts, or the polar alignment drifts.

HYBRID Telescope Control does not use a traditional pointing model. Instead, it uses a plate-solve-based approach where the Kalman filter serves as a self-calibrating, real-time pointing model. Every 5 seconds, a plate-solve measures the actual pointing position. The Kalman filter estimates the error. A SYNC command tells the mount where it actually is. And the pointing model updates itself — continuously, automatically, and without any user intervention.

The Traditional Approach: TPoint and Polynomial Models

Traditional mount modeling works by collecting a set of calibration points across the sky. At each point, the user slews to a known star, measures the offset between where the mount thinks it is pointing and where the star actually is, and records the error. After collecting enough points — typically 30 to 100 — a polynomial surface is fitted to the error data. This surface becomes the pointing model: a lookup table that corrects for systematic errors like polar alignment offset, tube flexure, gear periodic error, and atmospheric refraction.

TPoint, developed by Software Bisque, is the gold standard for this approach. It uses a sophisticated polynomial fitting algorithm that can achieve arcsecond-class pointing accuracy across the entire sky. Professional observatories use TPoint variants for their mount calibration. The approach is well-understood, well-proven, and well-documented.

Traditional pointing models are like a photograph of your mount's errors — accurate at the moment they were taken, but frozen in time. HYBRID's approach is like a live video — continuously updating, adapting to drift, and never going stale. Engineering philosophy

But polynomial models have inherent limitations. They are static: the model does not change after calibration. If the temperature shifts and the tube flexes differently, the model is wrong. If the polar alignment drifts over a long session, the model is wrong. If the mount is moved to a different location, the model must be rebuilt from scratch. The model captures spatial patterns — how pointing errors vary across the sky — but not temporal patterns — how pointing errors change over time.

HYBRID's Approach: The Kalman Filter as Pointing Model

HYBRID replaces the static polynomial model with a dynamic Kalman state vector that is continuously updated by plate-solve measurements. The Kalman filter estimates four quantities for equatorial mounts: RA position error, RA velocity error, Dec position error, and Dec velocity error. These four states capture the essential pointing dynamics — where the mount is off and how fast the error is changing.

For Alt-Az mounts, the state vector extends to six elements: Az position error, Az velocity error, Alt position error, Alt velocity error, and two bias terms that capture the nonlinear coupling between the axes. The Extended Kalman filter handles the coordinate transformation through a numerical Jacobian, allowing the same Kalman framework to work for both mount geometries.

The State Vector as Pointing Model

The Kalman state vector is not just a tracking correction — it IS the pointing model. The four (or six) state elements describe the mount's current pointing error in a compact, mathematical form. The state captures both the current offset and the rate of change, allowing the filter to predict where the mount will be in 5 seconds and correct before the error occurs.

This is fundamentally different from a polynomial model. A polynomial tells you "at RA 6h, Dec +30°, the mount is off by +12 arcseconds in RA." The Kalman state tells you "right now, at whatever position you are, the mount is off by +12 arcseconds in RA, and the error is increasing at +0.3 arcseconds per second." The Kalman model is position-independent and time-adaptive.

The SYNC Command: How the Model Updates

The SYNC command is the bridge between measurement and model. When a plate-solve determines the actual pointing position, the SYNC command tells the mount: "You are here now." OnStep (the mount controller) then recalculates its internal pointing model, correcting systematic errors.

ascom.sync_to_coordinates(ra, dec)
# Tells OnStep: "You are at this position NOW"
# OnStep then:
#   1. Updates its internal pointing model
#   2. Recalculates star catalog lookups
#   3. Adjusts tracking rates for the new position

In HYBRID's architecture, the SYNC command is issued after every Kalman correction cycle. The flow is: plate-solve captures the actual position → Kalman computes the error and the correction → the correction is applied as a MoveAxis command → the SYNC command updates the mount's internal model. This creates a closed-loop system where the pointing model is continuously refined by real measurements.

The SYNC command is the moment where the Kalman filter's mathematical model meets the mount's physical reality. Every SYNC is a course correction — a small adjustment that keeps the pointing model aligned with the real sky. Architecture note

SYNC vs. Traditional Calibration

A traditional calibration session involves slewing to specific stars, centering them in the eyepiece, recording the offset, and repeating 30-100 times. Each measurement is a static snapshot. The entire session takes 1-3 hours.

HYBRID's SYNC approach is continuous and automatic. Every plate-solve is effectively a calibration measurement — but instead of being recorded and processed later, it is applied immediately through the Kalman filter. The mount is continuously self-calibrating throughout the entire imaging session.

The Auto-Alignment System

HYBRID's auto-alignment system combines plate-solving with the Kalman filter to achieve polar alignment without user intervention. The approach is based on the TVC (Takahashi Variation Correction) method, adapted for plate-solve-based measurement.

Step 1: Initial Plate-Solve

At session start, HYBRID takes a short exposure (3-5 seconds), plate-solves the image to determine the exact pointing coordinates, and compares them to the expected position. The offset between expected and actual position reveals the mount's initial pointing error — a combination of polar alignment error, tube alignment error, and any mechanical offset.

Step 2: Iterative Convergence

The system then slews to a second star, plate-solves again, and computes the pointing error at the new position. By comparing the error vectors at two different positions, the system can separate polar alignment error (which produces a characteristic pattern of RA/Dec offsets across the sky) from other error sources.

The process repeats iteratively: slew, plate-solve, measure offset, compute correction, SYNC. Each iteration refines the alignment. After 3-5 iterations, the system converges to within ±5 arcseconds of polar alignment — comparable to what an experienced astronomer achieves with a dedicated polar scope and 30 minutes of manual adjustment.

AUTO-ALIGNMENT FLOW: Iteration 1: Plate-solve → Offset: (+45″ RA, +120″ Dec) → SYNC → Mount recalculates pointing model Iteration 2: Plate-solve → Offset: (+12″ RA, +28″ Dec) → SYNC → Model refined Iteration 3: Plate-solve → Offset: (+3″ RA, +5″ Dec) → Alignment complete — ±5 arcsec accuracy

Step 3: Continuous Refinement

After the initial alignment, the Kalman filter takes over. Every subsequent plate-solve further refines the pointing model. Over the course of a long imaging session, the pointing accuracy improves as the Kalman filter accumulates more data about the mount's behavior at different positions, temperatures, and hour angles.

Polar Alignment: The TVC Method

Polar alignment is the foundation of equatorial mount pointing. If the polar axis is not aligned with the celestial pole, every pointing model must compensate for the resulting field rotation — a systematic error that grows with hour angle and produces characteristic star trails in long exposures.

HYBRID implements a plate-solve-based polar alignment routine modeled on the TVC method. The algorithm measures the pointing offset at two positions in RA, computes the polar alignment error from the differential offset, and provides a correction vector that the user (or an automated focuser) can apply to the mount's azimuth and altitude adjusters.

PolarAlignment.Calculate():
  # Measure pointing at two RA positions
  solve_1 = plate_solve(exposure=3s)  # Current position
  sync_and_slew(ra_offset=+15min)    # Move 15 minutes in RA
  solve_2 = plate_solve(exposure=3s)  # Second position

  # Compute differential offset
  dRA  = solve_2.ra  - solve_1.ra
  dDec = solve_2.dec - solve_1.dec

  # TVC algorithm: extract polar alignment error

  polar_error_alt = g(dRA, dDec, latitude)

  return PolarCorrection(az_correction, alt_correction, accuracy=±5″)

How HYBRID Handles Common Error Sources

Every mount has a characteristic set of error sources. Here is how HYBRID's dynamic approach handles each one, compared to traditional static modeling.

Polar Alignment Error

Polar alignment error produces a field rotation that depends on hour angle. A static model captures this at the time of calibration. HYBRID's Kalman filter tracks it continuously — as the hour angle changes, the filter's Dec velocity state adapts to the changing field rotation rate.

Periodic Error (Worm Gear)

Periodic error is a sinusoidal wobble that repeats once per worm revolution. Traditional models capture the dominant harmonic. HYBRID's Kalman filter tracks the full periodic error waveform — including higher harmonics and amplitude variations that change with temperature and load.

Tube Flexure

Tube flexure changes with telescope orientation — pointing straight up produces different flexure than pointing at the horizon. A static model captures the average flexure. HYBRID's filter tracks flexure as a function of position, with the ML layer learning the mount-specific flexure pattern over time.

Atmospheric Refraction

Refraction bends starlight upward, with the amount depending on altitude above the horizon. At 30° elevation, refraction is 1.7 arcminutes; at 10°, it exceeds 5 arcminutes. HYBRID computes refraction in real-time based on the target's altitude and local atmospheric conditions, incorporating it into the Kalman state model.

Temperature-Dependent Drift

As temperature changes throughout the night, the telescope tube expands or contracts, shifting the focal plane and changing the optical alignment. A static model cannot adapt to this. HYBRID's Kalman filter tracks temperature-dependent drift as a slowly-varying bias state, adapting to thermal changes over the course of the session.

The Complete Alignment Architecture

Layer 1: Polar Alignment

TVC-based plate-solve alignment achieves ±5 arcsecond polar alignment accuracy. Applied once at session start. Refined continuously by the Kalman filter.

Layer 2: Pointing Model

Kalman state vector serves as the pointing model. Updated every 5 seconds by plate-solve measurements. Self-calibrating, drift-adaptive, zero manual intervention.

Layer 3: SYNC Correction

After each plate-solve, SYNC tells the mount where it actually is. OnStep recalculates its internal model. The mount's physical pointing model stays current.

Layer 4: ML Enhancement

The ML layer learns mount-specific patterns — periodic error harmonics, flexure curves, thermal drift profiles. Improves prediction accuracy over time.

When to Use Traditional Mount Modeling

HYBRID's dynamic approach is not always the best choice. Traditional mount modeling has specific advantages in certain scenarios:

  • Fixed observatory installations — If the mount never moves, a TPoint calibration can be done once and used for years. The static model is perfectly adequate for a permanently mounted system.
  • Sub-arcsecond pointing requirements — For applications requiring pointing accuracy better than ±1 arcsecond (such as fiber-fed spectroscopy), TPoint's polynomial surface with 100+ calibration points achieves accuracy that plate-solve-based approaches cannot match.
  • Non-ASCOM mounts — HYBRID's SYNC-based approach requires ASCOM compatibility. For mounts without ASCOM drivers, traditional model-based correction may be the only option.
The best pointing model is the one that matches your use case. For portable, unattended, automated imaging — the Kalman approach. For fixed observatories requiring sub-arcsecond pointing — TPoint. For everything in between — HYBRID's hybrid approach gives you the best of both worlds. Design principle

Lessons Learned

1. The Model IS the Filter

The Kalman filter is not just a tracking correction tool — it IS the pointing model. The state vector captures everything a polynomial model captures (position-dependent error) plus everything a polynomial model misses (temporal drift, rate of change, adaptive noise). The insight that the Kalman state vector replaces the polynomial model was the key architectural decision.

2. Continuous Calibration Beats Periodic Calibration

A traditional pointing model is calibrated once and goes stale. HYBRID's model is calibrated continuously. Every plate-solve is a calibration measurement. Every SYNC is a model update. The result is a pointing model that never goes stale — it is always current, always adapted to the present conditions.

3. The SYNC Command is Underappreciated

The ASCOM SYNC command is one of the most powerful operations in telescope control. It updates the mount's internal pointing model instantly, correcting systematic errors without requiring a full recalibration. HYBRID leverages SYNC as the primary mechanism for keeping the pointing model current.

4. Polar Alignment is the Foundation

No amount of software correction can fully compensate for poor polar alignment. The TVC-based auto-alignment system achieves ±5 arcsecond accuracy, which provides the foundation for the Kalman filter to build upon. Better polar alignment means better initial pointing, which means faster convergence and better tracking from the first frame.
