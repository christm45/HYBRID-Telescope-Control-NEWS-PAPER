Phase 0: Trust the Math Before You Change It

Before touching a single line of the production Kalman or Extended Kalman filter…

None of this changed behavior.

Phase 1: Acceleration States and Multi-Star Fusion

The 4-state linear Kalman filter and the 6-state Extended Kalman filter both gained acceleration terms.

Multi-star tracking landed as a pre-filter fusion step.

The trickiest part of this phase wasn't the math — it was backward compatibility.

Phase 2: Noise That Adapts Per Axis

Before this phase, the ML noise-tuning layer applied one combined scale to every state alike.

The point of building the acceleration state in Phase 1 wasn't the state itself. It was giving Phase 2 something honest to scale noise against. Engineering note

Phase 3: Teaching the ML Layer to Remember

Feature expansion: cloud cover is now fully wired.

Model versioning with automatic rollback.

Cross-mount transfer learning.

“Online learning” got the honest treatment.

Phase 4: The Fusion Gain Stops Being a Fixed Number

Multi-target feature sharing indexes training history by declination band.

Adaptive gain scheduling replaced a fixed 70/30 Kalman/ML blend.

An explainability layer surfaces ML.NET's permutation feature importance.

Phase 5: What “Real-Time” Means Over Plain HTTP

ASCOM Alpaca is a REST protocol, not WebSocket.

A rate-smoothing filter and a predictive sync-timing scheduler both landed as small, pure, standalone functions.

Phase 6: The Password Was Still “telescope”

The security phase folded in a real finding from the very first research pass.

Audit logging reused the project's existing structured logger.

The real surprise of this phase wasn't in the security code at all.

What Got Deliberately Left Undone

Not everything got wired all the way through, and that's written down rather than hidden.

The pattern repeats across every phase.

Lessons Learned

A Flaky Test Is Still a Bug

One isolation test in the EKF suite asserted exact equality between two separately constructed filter instances.

The Pipeline Is Part of the Test Suite

A comprehensive test suite that CI never runs is a false sense of security.
